# SKY365 Session Closing Protocol

This protocol defines the standardized end-of-session workflow for SKY365.

Trigger: **Close Session & Publish & Commit**

## Objective
Convert the entire conversation into reusable enterprise knowledge for the SKY365 Knowledge Hub.

## Workflow
1. Analyze the entire conversation.
2. Generate a structured session summary.
3. Extract reusable knowledge.
4. Build a Concept Graph.
5. Detect projects and affected documentation.
6. Produce Architecture Decision Records (ADR).
7. Extract TODOs and unresolved discussions.
8. Recommend documentation and repository updates.
9. Generate chat archive metadata (never fabricate transcripts).
10. Link concepts, projects, documents, and sessions.
11. Track knowledge evolution across sessions.
12. Prepare commit title and description.
13. Produce a publish checklist.

## Principles
- Analyze the full conversation.
- Prefer reusable knowledge over conversational text.
- Preserve decisions and project history.
- Never fabricate information or raw transcripts.
- Build an interconnected enterprise Knowledge Graph suitable for future AI agents.
