# Sky365 AI Study Agent — MVP

## Strategic decision

Sky365 should prioritize one focused education product before expanding into virtual labs, humanoid robotics, or a full LMS.

The first product is:

> **Sky365 AI Study Agent**

An AI-native learning layer that converts existing educational content into personalized study experiences.

## Core user flow

1. A student uploads or connects a PDF, video, lecture, link, or course material.
2. The system analyzes the content and builds a structured knowledge map.
3. The agent adapts explanations to the student's level and preferred learning style.
4. It generates:
   - personalized explanations
   - flashcards
   - quizzes and exam questions
   - study guides
   - summaries
   - podcast-style audio lessons
   - study plans
5. It measures performance, identifies weak areas, and regenerates explanations and practice accordingly.

## Product principle

This is not another LMS and not merely a chatbot over documents.

The product must become a measurable learning loop:

> Content → Personalization → Practice → Assessment → Weakness detection → Re-teaching

## MVP scope

### Required in version one

- User authentication
- Course or subject workspace
- PDF and text ingestion
- Video transcript ingestion where available
- AI-generated summaries
- Flashcards
- Multiple-choice and short-answer quizzes
- Study guide generation
- Personalized study plan
- Student performance tracking
- Weak-topic detection
- Adaptive re-explanation
- Arabic and English support

### Optional after validation

- Podcast and audio lesson generation
- Teacher dashboard
- Classroom and cohort analytics
- LMS integrations
- Curriculum mapping
- Virtual-lab integration
- Robotics integration
- Parent dashboard
- University administration features

## 30-day execution target

### Days 1–7: Product definition

- Assign one accountable Product Owner
- Select the first target user segment
- Define the primary learning outcome
- Review all existing Sky365 education ideas and code
- Freeze nonessential education initiatives
- Produce a clickable or functional prototype

### Days 8–14: MVP build

- Implement content ingestion
- Build knowledge extraction and chunking
- Implement flashcards, quizzes, summaries, and study guides
- Add basic student profile and progress state
- Create the adaptive feedback loop

### Days 15–21: Pilot preparation

- Recruit a small group of real students
- Prepare two or three complete learning journeys
- Add analytics for activation, completion, return rate, and assessment improvement
- Fix critical usability failures

### Days 22–30: Live pilot

- Launch to real students
- Observe usage directly
- Interview users
- Measure repeat usage and learning improvement
- Decide whether to continue, narrow, or reposition

## Metrics that matter

Do not judge the MVP by the number of features.

Track:

- Percentage of users who complete the first study session
- Percentage who return within seven days
- Number of study sessions per active user
- Quiz-score improvement after adaptive re-teaching
- Time saved creating study materials
- Percentage willing to pay or recommend
- Most frequently abandoned step

## Competitive reference points

The product may learn from:

- Klar Labs / KlarOS
- CENTURY Tech
- Sana
- Seneca Learning
- Sparx Learning
- Labster
- VLaby
- PraxiLabs

Sky365 should not copy their interfaces. The objective is to combine adaptive learning, content transformation, measurable outcomes, and later integrations with labs and embodied AI.

## Strategic expansion path

After product-market validation:

1. Add teacher and institution workflows.
2. Integrate virtual labs such as VLaby or PraxiLabs.
3. Add curriculum and assessment mapping.
4. Add institutional analytics and governance.
5. Integrate humanoid or educational robots as a physical interaction channel.
6. Position Sky365 as the orchestration layer across content, learning agents, labs, and robotics.

## Critical operating truth

Sky365 does not lack ideas. It lacks ruthless prioritization around one education product, one owner, one user segment, and one launch date.

The immediate goal is not to build the complete education ecosystem.

The immediate goal is to prove that students repeatedly use the Sky365 AI Study Agent and measurably learn better with it.
