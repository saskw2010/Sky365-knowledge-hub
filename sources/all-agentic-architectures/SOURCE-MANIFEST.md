## Migration policy

### Eligible for migration

- HTML documentation portals and subapplications.
- CSS, JavaScript, images, icons, diagrams, and motion assets required to render those portals.
- Markdown and HTML documentation relevant to Sky365 architecture, education, strategy, research, and product decisions.
- Selected upstream documentation used as reference material, with explicit attribution.

### Not eligible for this documentation migration

- Python package source code.
- Runnable notebooks.
- Tests and CI for the upstream Python package.
- Packaging and runtime implementation files.

## Required provenance fields

Each migration record must contain:

- Source repository.
- Source commit or snapshot date.
- Original path.
- Destination path.
- Content owner or author.
- License.
- Transformation state: `unchanged`, `adapted`, `summarized`, or `extended`.
- Link-validation status.
- Visual-validation status.

## Canonical rule

After a page is migrated, validated, and promoted, future Sky365 documentation edits occur in `Sky365-knowledge-hub`. The source fork remains available for implementation reference, provenance, and history.
